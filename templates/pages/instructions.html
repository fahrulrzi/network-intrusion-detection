üì° Untuk Analisis Network Traffic Pribadi

Untuk menggunakan sistem ini dengan data network traffic pribadi, Anda perlu menginstall network traffic analyzer terlebih dahulu:

Step 1: Install Network Traffic Capturer

Pilih salah satu tools berikut:

  • Zeek (Bro) - Recommended untuk analisis mendalam 
    # Ubuntu/Debian 
sudo apt-get install zeek

# CentOS/RHEL 
sudo yum install zeek
  • Wireshark - GUI-based network analyzer
  • tcpdump - Command-line packet analyzer
  • nfcapd - NetFlow capture daemon

Step 2: Capture Network Traffic

# Contoh menggunakan Zeek 
zeek -i eth0 local

# Contoh menggunakan tcpdump
sudo tcpdump -i eth0 -w capture.pcap        

# Contoh menggunakan Wireshark (GUI)
# File -> Capture -> Start capturing on interface

Step 3: Extract Features

Ekstrak fitur-fitur berikut dari hasil capture menggunakan tools analisis:

Basic Features:
  • ‚Ä¢ Protocol (proto)
  • ‚Ä¢ Service (service)
  • ‚Ä¢ Connection state (state)
  • ‚Ä¢ Packet counts (spkts, dpkts)
  • ‚Ä¢ Byte counts (sbytes, dbytes)
Advanced Features:
  • ‚Ä¢ Timing features (TTL, jitter)
  • ‚Ä¢ TCP window sizes

üìã Deskripsi Fitur

Sistem menggunakan 41 fitur yang telah diseleksi dari 47 fitur total dalam dataset UNSW-NB15. Berikut adalah penjelasan detail setiap fitur:

üîó Basic Connection Information

proto (Protocol)

Protokol transport yang digunakan dalam koneksi.
Contoh: tcp, udp, unas, arp, ospf, irtp, 3pc, mhrp, rdp

service (Service)

Layanan jaringan yang digunakan.
Contoh: http, dns, smtp, ftp, ssh, ssl, snmp, atau "-" jika tidak teridentifikasi

state (Connection State)

Status koneksi saat ini.
Nilai: FIN, INT, CON, REQ, ACC, RST, CLO

rate (Connection Rate)

Jumlah paket per detik dalam koneksi.
Satuan: packets/second

üì¶ Packet Information

spkts / dpkts

Jumlah paket source‚Üídest (spkts) dan sebaliknya (dpkts).

sbytes / dbytes

Jumlah bytes source‚Üídest (sbytes) dan sebaliknya (dbytes). Satuan: Bytes

smean / dmean

Rata-rata ukuran paket source (smean) dan destination (dmean). Satuan: Bytes/paket

sload / dload

Kecepatan transfer data source (sload) dan dest (dload). Satuan: bits/s

‚è±Ô∏è TTL & Timing Information

sttl / dttl

TTL paket dari source (sttl) dan destination (dttl). Untuk anomali routing dan OS fingerprinting.

sinpkt / dinpkt

Waktu antar kedatangan paket untuk source dan destination. Satuan: ms

sjit / djit

Jitter (variasi waktu) paket source dan destination. Satuan: ms

dloss

Jumlah paket yang hilang/retransmit pada destination. Indikator kualitas koneksi.

üîå TCP Information

swin / dwin

TCP window size advertisement untuk source dan destination.

stcpb / dtcpb

TCP base sequence number untuk source dan destination.

tcprtt

TCP setup round-trip time (synack + ackdat). Satuan: detik

synack / ackdat

Waktu setup TCP: SYN‚ÜíSYN_ACK (synack) dan SYN_ACK‚ÜíACK (ackdat). Satuan: detik

üåê HTTP & Application Features

trans_depth

Kedalaman pipeline dalam HTTP request/response. Deteksi HTTP pipelining abuse.

response_body_len

Ukuran uncompressed content dari HTTP server response. Satuan: Bytes

ct_flw_http_mthd

Jumlah flow dengan HTTP methods (GET, POST, dll). Deteksi HTTP method abuse.

is_ftp_login / ct_ftp_cmd

FTP login status (1/0) dan jumlah FTP commands. Deteksi FTP brute force/abuse.

üìä Connection Count Features

Fitur-fitur berikut menghitung jumlah koneksi dengan karakteristik tertentu dalam 100 koneksi terakhir:

ct_srv_src / ct_srv_dst

Koneksi dengan service dan source/destination address yang sama

ct_dst_ltm / ct_src_ltm

Koneksi dengan destination/source address yang sama

ct_src_dport_ltm

Koneksi dengan source address dan destination port yang sama

ct_dst_sport_ltm

Koneksi dengan destination address dan source port yang sama

ct_dst_src_ltm

Koneksi dengan source dan destination address yang sama

ct_state_ttl

Koneksi dengan state dan TTL range yang sama

üìà Pentingnya Count Features: Fitur-fitur ini penting untuk mendeteksi pola serangan seperti port scanning, DDoS, dan reconnaissance activities.

‚úÖ Boolean Features

is_sm_ips_ports

Menunjukkan apakah source dan destination IP address serta port numbers sama.

Nilai: 1 = Ya (sama), 0 = Tidak (berbeda)
Penggunaan: Deteksi loopback attacks dan self-connection anomalies
